# 双影密探 ShadowSleuth 项目经验总结

> 本文档从 0 到 v1.2.0 的迭代过程中，总结需求变化、技术决策、踩坑记录与可迁移的项目经验。

---

## 1. 项目背景

用户希望开发一款**纯本地离线**的 Android 重复图片查找工具，核心诉求：

- 不上传、不联网、不自动删除。
- 快速扫描本地图片并按规则找出重复项。
- 界面简洁、专业，操作直接。
- 最终交付可安装的 APK，并维护 GitHub 仓库。

技术栈最终确定为：Kotlin + Jetpack Compose + Material 3 + MediaStore + Coil。

## 3. 关键决策与原因

### 3.1 为什么只匹配文件名和文件大小？

- **文件名**：直观，同名图片通常是重复下载/保存。
- **文件大小**：相同字节大小的图片极大概率是重复副本。
- **分辨率**：尝试过但误报率高，已移除。
- **保存时间**：用户反馈保存时间完全一致不应算重复，已作为排除规则。

### 3.2 为什么使用 MediaStore 而不是直接遍历文件？

- 不需要 `MANAGE_EXTERNAL_STORAGE` 即可读取大部分图片。
- 扫描速度快，只读元数据，不加载原图。
- 兼容 Android 10+ 分区存储。

### 3.3 为什么默认深色主题？

- 用户希望专业、神秘风格。
- 深色主题在相册整理场景下视觉疲劳更低。

### 3.4 为什么手动扫描而不是自动扫描？

- 用户明确反馈：进入 APP 后不要自动扫描，避免不必要的资源占用。
- 手动触发更符合用户控制感。

### 3.5 为什么 Android 10+ 删除仍可能需要二次授权？

- 这是 Android 系统的分区存储限制，普通应用无法绕过。
- 折中方案：提供 `MANAGE_EXTERNAL_STORAGE` 开关，用户开启后可一次授权。
- 保留 `RecoverableSecurityException` 作为降级方案。

## 4. 踩坑记录

| 阶段 | 问题 | 解决方案 |
|------|------|----------|
| 早期 | 拖动条无法拖动 | 状态改为 `MutableStateFlow` |
| 早期 | 搜索页选择图片闪退 | 兼容 Android 10+ DATA 列缺失 |
| 中期 | 长按删除提示权限不足 | 捕获 `RecoverableSecurityException` 并弹系统授权框 |
| 中期 | 结果出现 `0 × 0` 图片 | 扫描时过滤无效分辨率 |
| 中期 | 纬度经度读取空指针 | 安全取值 `latLong?.let { ... }` |
| 后期 | 主题切换状态不持久 | 使用 DataStore 持久化 |
| 后期 | 权限申请不够大胆 | 启动时请求 WRITE + 提供 MANAGE_EXTERNAL_STORAGE 入口 |

## 5. 可迁移经验

1. **Compose 状态管理**：用 `StateFlow` 而不是普通 `var`，UI 用 `collectAsState()` 订阅。
2. **权限模型要早规划**：Android 10+ 的分区存储对删除类应用影响大，需要在设计阶段就考虑。
3. **MediaStore 是只读元数据的利器**：适合扫描、统计、分组类工具，不加载原图。
4. **用户说「做减法」时要果断**：分辨率匹配虽然容易实现，但效果差就移除。
5. **文档与代码同步更新**：每次版本变更后同步更新 README、架构文档和技能文档，降低后续维护成本。
